API Documentation

Everything you need to integrate VerityAI's detection engine.

Authentication

All API requests must be authenticated using a Bearer token in the Authorization header. You can find your API key on the homepage.

Authorization: Bearer YOUR_API_KEY

Endpoints

1. Analyze Text

Endpoint to analyze a plain text string for fake news indicators.

POST /v1/analyze

Request Body:

{
  "text": "Your news article text goes here..."
}

Response (200 OK):

{
  "credibility_score": 85,
  "verdict": "Likely Credible",
  "sources_checked": 8,
  "matches_found": 5,
  "analysis_time_ms": 3200,
  "findings": [
    {
      "title": "Credibility Indicators Found",
      "description": "Positive indicators detected: 'research shows', 'peer-reviewed'."
    }
  ]
}

Error Codes

VerityAI uses standard HTTP response codes to indicate the success or failure of an API request.

  • 200 OK - Request successful.
  • 400 Bad Request - The request body was invalid or missing the text parameter.
  • 401 Unauthorized - Invalid or missing API key.
  • 429 Too Many Requests - You have exceeded your rate limit (default 100 requests/minute).
  • 500 Internal Server Error - Something went wrong on our end.
